# LibreNMS L2 Topology → Mermaid Diagram

Небольшой Python-инструмент для автоматической визуализации **L2-топологии сети (CDP / LLDP)** на основе данных из **LibreNMS API**.
Скрипт собирает информацию о соседях устройств, очищает её от лишних endpoint-устройств (телефоны, принтеры и т.п.) и генерирует **Mermaid Flowchart**, который можно отобразить в Markdown, Mermaid Live или документации.

Проект ориентирован на сетевых администраторов и инженеров, которым нужно быстро получить **наглядную карту сетевых соединений** без ручного рисования.

---

# Возможности

* получение L2-линков из LibreNMS API
* разрешение `device_id → hostname`
* разрешение `port_id → interface`
* дедупликация линков (A↔B)
* удаление "сквозных" CDP-линков, если есть LLDP
* исключение устройств через конфигурационный файл
* статистика типов endpoint-устройств
* collapse параллельных линков (uplink bundles / LAG)
* генерация диаграммы **Mermaid Flowchart**
* Junction-узлы (●), позволяющие отображать **порты на обеих сторонах линка**

---

# Пример диаграммы

Сгенерированная схема выглядит примерно так:

```
[CORE-SW1] ──xe-0/0/1── ● ──xe-1/0/1── [DIST-SW1]
      │
      └──────xe-0/0/2── ● ──Gi1/0/48── [ACCESS-SW1]
```

Где:

* прямоугольники — сетевые устройства
* ● — точка соединения (кабель)
* подписи — интерфейсы

# Настройка

В .env необходимо указать параметры API:

```python
LIBRENMS_URL = "https://librenms.example.com"
API_TOKEN = "YOUR_API_TOKEN"
```

API-токен можно создать в LibreNMS:

```
LibreNMS → Settings → API
```

---

# Запуск

Запуск выполняется обычным образом:

```bash
python L2-mermaid-from-librenms.py
```

Скрипт:

1. собирает устройства, порты и линки из LibreNMS
2. очищает топологию
3. печатает Mermaid-код диаграммы

Пример вывода:

```
flowchart LR

SW_CORE1["SW-CORE1"]
SW_DIST1["SW-DIST1"]

J0@{shape: f-circ}
SW_CORE1 --|Gi1/0/1|--- J0
J0 --|Gi1/0/48|--- SW_DIST1
```

# Исключение устройств (exclude_hosts.conf)

Чтобы убрать из диаграммы лишние устройства (телефоны, серверы, принтеры и т.п.), используется файл:

```
exclude.conf
```

Формат:

```
# комментарии начинаются с #

name=test-server
hardware=Cisco IP Phone
hardware=Polycom
hardware=HP Printer
```

Поддерживаемые ключи:

| ключ       | значение            |
| ---------- | ------------------- |
| `name`     | hostname устройства |
| `hardware` | тип устройства      |

Если правило совпадает, соответствующий линк удаляется из топологии.

При этом скрипт пишет сообщения в консоль:

```
Excluded by hardware=Cisco IP Phone:
SW-ACCESS1 Gi1/0/10 -> SEP001122334455 Port1
```

# Статистика endpoint-устройств

Перед применением исключений скрипт печатает статистику типов устройств:

```
Endpoint hardware statistics:

 340  Cisco IP Phone 8845
 120  Polycom VVX 501
  70  HP LaserJet
  35  VMware ESXi
```

Это помогает быстро определить, какие устройства стоит добавить в `exclude_hosts.conf`.

# Collapse параллельных линков

Если между двумя устройствами несколько линков (например uplink bundle), скрипт может объединить их:

```
SW1 ──3 links── ● ── SW2
```

Это значительно уменьшает размер диаграммы и делает её более читаемой.

Опция включается глобальной переменной:

```python
COLLAPSE_PARALLEL_LINKS = True
```

# Ограничения

* диаграмма показывает **только L2 соседство**
* LLDP/CDP данные зависят от корректной настройки устройств
* LLDP может проходить через транки Cisco при native vlan 1, создавая "ложные" соедиенения  
* очень большие сети могут требовать дополнительной фильтрации
